    Web安全知识

    一、组织及资源

    • OWASP(Open Web Application Security Project,开放式Web程序安全项目) —— https://owasp.org

    二、风险与攻击

    对于Web程序的安全问题，一个首要的原则是：**永远不要相信你的用户**
    1. 注入攻击
      • 系统命令(OS Command)注入
      • SQL(Structured Query Language,结构化查询语言)注入(SQL Injection)可参考https://www.owasp.org/index.php/SQL_Injection
         A、 攻击原理：如果直接将用户传入的数据作为参数使用字符串拼接的方式插入到SQL查询语句中，那么攻击者可以通过注入其他语句来执行攻击操作，这些操作包括可以通过SQL语句做的任何事情。
         B、 攻击举例(其中[]内的内容为用户传入参数)：
          （SELECT * FROM students WHERE id = [111 or 1=1 –] ）
          （SELECT * FROM students WHERE id = [111 ;drop table user;] ）
         C、 防范措施：
        • 使用ORM可以一定程度上避免SQL注入问题
        • 验证输入类型
        • 参数化查询，在构造SQL语句时避免使用拼接字符串或字符串格式化的方式来构建SQL语句。而要使用各类接口库提供的参数化查询方法。
        • 转义特殊字符，比如引号、分号和横线等。
      • NoSQL注入
      • ORM(Object Relational Mapper,对象关系映射)注入
    2. XSS(Cross Site Scripting,跨站脚本)攻击(避嫌CSS)
      A、 攻击原理：
        XSS是注入攻击的一种，攻击者将代码注入被攻击这的网站中，用户一旦访问网页便会执行被注入的恶意脚本，其主要分为反射型XSS攻击(Reflected XSS Attack)h和存储型XSS攻击(Stored XSS Attack)
      B、攻击示例：
        反射型攻击（又称为非持久型XSS），这种攻击会通过URL注入攻击脚本，只有当用户访问这个URL时才会执行攻击脚本
      (http://example.com/hello?name=[])攻击者通过JavaScript脚本几乎可以做任何事情：窃取用户的cookie和其他敏感数据，重定向到钓鱼网站，发送其他请求，发布广告信息，关注用户等，      一般通过点击不明链接。
        存储型XSS也被称为持久型XSS，它会把攻击代码存储到数据库中，例如用户留言中加入恶意JavaScript代码
      C、主要防范措施
        a、HTML转义
      防范XSS攻击最主要的方法是对用户输入的内容进行转义，转义后可以确保用户输入的内容在浏览器中作为文本显示，而不是作为代码解析
      可以使用JinJa2提供的escape（）函数
        b、用户输入类型验证，其中的url输入为重（<a href=[“javascript:alert(‘Bingo!’);”>Website]）
    3. CSRF(Cross Site Request Forgery,跨站请求伪造)攻击（又称为One-Click Attack或者Session Riding）
       A、攻击原理：
        攻击者利用用户在浏览器中保存的认证信息，项对应站点发送伪造请求。
       B、攻击示例：用户登录A站，然后访问B站（攻击者），B站中隐藏自执行表单，由于已经将A站的登录信息保存在cookie中。故可欺骗A站。
       C、防范措施：
        a. 正确使用HTTP方法（防范CSRF的基础就是正确使用HTTP方法）GET方法属于安全方法，不会改变资源状态，仅用于获取资源，因此又被称为幂等方法（idempotent method）
        b. CSRF令牌效验
      当处理非GET请求时，要想避免CSRF攻击，关键在于判断请求是否来自自己的网站。但容易伪造，故可以在客户端页面中加入伪随机数。
      具体来说，服务器创建伪随机数添加到表单中的隐藏字段里和session变量里，并且设定一定时间过期

    2、SSL证书

    2.1、概念及名词

    • CSR(Certificate Signing Request) 证书注册请求
    • SSL(Secure Socket Layer) 安全套接字层，位于可靠的面向连接的网络层协议和应用层协议之间的一种协议；该协议由两层组成：SSL记录协议和SSL握手协议
    • TLS(Transport Layer Security) 传输层安全协议，用于两个应用程序之间提供保密性和数据完整性；该协议由两层组成: TLS记录协议和TLS握手协议

      2.2、Generate a CSR using the openssl utility

      2.2.1、Generate a private key:

      openssl genrsa -out client.key

    2.2.2、Generate the CSR:

    openssl req -out client.csr -key client.key -new
